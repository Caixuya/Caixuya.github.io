我博客的图片都是上传到github上的，遇到过打不开自己博客图片的情况，为了方便大家看图片，这里就详细说一下解决github图片加载不成功的办法，分为linux端和windows端，本质都是一样的，都是修改hosts文件

windows端

  • 找到hosts文件，路径如下:c:\windows\system32\drivers\etc

  • 因为hosts文件是系统文件，不可以直接修改，windows和linux都是一样的，都要先设置权限，另外如果widows系统目录不是C盘，也应注意把盘符改成系统所在的盘符,原因刚刚也解释啦，hosts是系统文件

  • 找到hosts文件后先设置权限，或者直接复制粘贴到桌面，等修改好后替换掉原来的hosts文件，这里附上前者的（设置权限）教程

  • 右键“属性”

  • 点击“安全”，给用户赋予完全控制权限，我这里是给users赋予完全控制权，

修改内容

  • 赋权后就可以修改文件了，在hosts原文件最后面添加以下内容，ctrl+s保存，回到浏览器刷新一下页面，就可以看到原本不能显示的图片又刷刷刷回来啦

linux端

  • ctrl+alt+t调出终端
  • 输入以下代码给hosts文件赋权

    sudo chmod 777 /etc/hosts

敲完命令后没有报错就是成功的，像这样

  • 用gedit或者vim/vi修改文件

    用gedit修改文件

  • gedit安装，在终端输入以下命令进行安装

    ubuntu，deeping系列的linux系统安装命令

    sudo apt-get install gedit

centos系列的安装命令

sudo yum install gedit

视自身情况选其中一个

  • 安装完成后使用gedit的命令如下：

    gedit /etc/hosts

输入命令会打开这样一个编辑窗口

  • 直接复制粘贴上面的内容（右边目录点击“修改内容”即可跳转），添加的内容是一样的，在此就不赘述啦，然后ctrl+s保存，点击右上角红色小叉叉直接退出即可，退出后会返回终端，没有报错即修改成功，就像最后面的那张图那样（右边目录点击“大功告成”即可跳转）

    用vim/vi修改文件

  • vim的安装（如果系统没有vim）
  • ubuntu和deeping安装和gedit的没啥区别，直接把上面安装代码中的gedit改成vim即可
  • centos系列的安装有一点点不同，需要输入以下命令：

    sudo yum -y install vim

  • 安装完成后使用vim/vi的命令如下：

    vi /etc/hosts

vim /etc/hosts

两个命令二选一，都可以，打开界面如下：

  • 打开界面后输入键盘”i”键

    i

  • 最下角会有”插入“两个字，这样就是进入插入模式了，然后在最末尾处把上面的内容（右边目录点击“修改内容”即可跳转）添加进去

  • 添加完后按esc键退出编辑模式，再键入”:wq“

    :wq

  • 这时左下角会出现你键入的命令

  • (注意冒号也要键入，键入英文模式下的冒号），这是保存并退出的命令
