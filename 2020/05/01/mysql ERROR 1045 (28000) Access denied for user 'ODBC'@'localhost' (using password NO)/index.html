mysql ERROR 1045 (28000) Access denied for user 'ODBC'@'localhost' (using password NO)

在cmd中敲入mysql后，出现下面的错误，找了很久没有找到解决方案

mysql ERROR 1045 (28000) Access denied for user ‘ODBC‘@’localhost’ (using password NO)

在找答案的过程中受一位网友的启发，翻译了这句话，尝试了一下，惊喜地发现困扰了我好几天的问题解决了

不过好像这个问题解决和不解决都没什么关系，我只是单纯强迫症犯了和这个问题刚起来了

这句话翻译过来就是你的mysql没有一个不用输入密码就可以登录的ODBC用户，解决方案就是：创建一个没有密码的ODBC用户，如下所示：

CREATE USER 'ODBC'@'localhost' IDENTIFIED BY '';
然后更新一下权限
flush privileges;

说到创建用户，版本不同命令是有些微差别的，如果上面这条命令不适用你的mysql，就去找找其他的创建用户的命令，这个问题的解决方案就到这里啦～～
附上图片：

另外附上mysql登录命令：mysql -u root -p;
关闭命令：service mysqld stop;
修改密码命令：set password for 'root'@'localhost'=password('991007');
赋予所有权限：grant all privileges on *.* to 'root'@'%' identified by 'root';
更新权限命令update mysql.user set Grant_priv='Y',Super_priv='Y' where user = 'root' and host = '%';
