windows端

  • 找到hosts文件，路径如下:c:\windows\system32\drivers\etc
  • 因为hosts文件是系统文件，不可以直接修改，windows和linux都是一样的，都要先设置权限，另外如果widows系统目录不是C盘，也应注意把上面的路径中的C:改成系统所在的盘符,原因刚刚也解释啦，hosts是系统文件
  • 找到hosts文件后先设置权限，或者直接复制粘贴到桌面，等修改好后替换掉原来的hosts文件，这里附上前者的（设置权限）教程
  • 找到hosts文件后，右键选择“属性”

  • 点击“安全”，给用户赋予完全控制权限，我这里是给users赋予完全控制权，

修改内容

linux端

  • 找到hosts文件，路径如下/etc/hosts，如果你的hosts不在这里，那么输入这条命令查询whereis hosts，要找的是hosts文件，其他hosts.*** 的不是我们要找的文件
  • ctrl+alt+t调出终端
  • 输入以下代码给hosts文件赋权

    sudo chmod 777 /etc/hosts

敲完命令后没有报错就是成功的，像这样

  • 用gedit或者vim/vi修改文件

    用gedit修改文件

  • gedit安装，在终端输入以下命令进行安装

    ubuntu，deeping系列的linux系统安装命令

    sudo apt-get install gedit

centos系列的安装命令

sudo yum install gedit

视自身情况选其中一个

  • 安装完成后使用gedit编辑文件的命令如下：

    gedit /etc/hosts

输入命令会打开这样一个编辑窗口

  • 直接复制粘贴上面的内容（右边目录点击“修改内容”即可跳转），添加的内容是一样的，在此就不赘述啦，然后ctrl+s保存，点击右上角红色小叉叉直接退出即可，退出后会返回终端，没有报错即修改成功，就像最后面的那张图那样（右边目录点击“大功告成”即可跳转）

    用vim/vi修改文件

  • vim的安装（如果系统没有vim）(linux系统自带vi)
  • ubuntu和deeping安装和gedit的没啥区别，直接把上面安装代码中的gedit改成vim即可
  • centos系列的安装有一点点不同，需要输入以下命令：

    sudo yum -y install vim

  • 安装完成后使用vim/vi编辑文件的命令如下：

    vi /etc/hosts

    vim /etc/hosts

两个命令二选一，都可以，打开界面如下：

  • 打开界面后输入键盘”i”键

    i

  • 拉到终端最底端，左下角会有”插入“两个字，这样就是进入插入模式了，然后在最末尾处把上面的内容（右边目录点击“修改内容”即可跳转）添加进去

  • 添加完后按“esc”键退出编辑模式，再键入”:wq“

    :wq

  • 键入时左下角会出现你键入的命令，然后回车即可

  • (注意冒号也要键入，键入英文模式下的冒号），这是保存并退出的命令

    大功告成

  • 像下图这样没有出现报错就是大功告成啦

Markdown是一种轻量级的「标记语言」

Markdown是一种可以使用普通文本编辑器编写的标记语言，通过简单的标记语法，它可以使普通文本内容具有一定的格式。它允许人们使用易读易写的纯文本格式编写文档，然后转换成格式丰富的HTML页面，Markdown文件的后缀名便是“.md”

MdEditor是一个在线编辑Markdown文档的编辑器

MdEditor扩展了Markdown的功能（如表格、脚注、内嵌HTML等等），以使让Markdown转换成更多的格式，和更丰富的展示效果，这些功能原初的Markdown尚不具备。

Markdown增强版中比较有名的有Markdown Extra、MultiMarkdown、 Maruku等。这些衍生版本要么基于工具，如Pandoc，Pandao；要么基于网站，如GitHub和Wikipedia，在语法上基本兼容，但在一些语法和渲染效果上有改动。

MdEditor源于Pandao的JavaScript开源项目，开源地址Editor.md，并在MIT开源协议的许可范围内进行了优化，以适应广大用户群体的需求。向优秀的markdown开源编辑器原作者Pandao致敬。
