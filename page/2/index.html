递归函数–递归，尾递归

函数在内部可以调用其他函数，也可以自己调用自己，自己调用自己的函数叫递归函数。

例子一：

如：

python
def fact(n):
    if n==1:
        return 1        
     #n * fact(n-1)中，当n=1时(n-1)=0，为了避免n=1时出错，直接让n=1时返回1
    return n * fact(n-1)

假设这里n=5,即 def fact(5)：

这个函数的执行过程是：先调用fact函数的返回值：n * fact(n-1)

fact(5) ==>n * fact(n-1)，==> 5 * fact(5 - 1) ==> 5 * fact(4)

这里fact（4）再次调用fact函数，即将算式中的fact(4)替换成返回值

fact(4) ==>n * fact(n-1)，==>5 * (4 * fact(4 - 1)) ==> 5 * (4 * fact(3))

以此类推，直到算式中没有fact()函数

fact(3) ==>n * fact(n-1)，==>5 * (4 * (3 * fact(3-1))) ==> 5 * (4 * (3 * fact(2)))

fact(2) ==>n * fact(n-1)，==>5 * (4 * (3 * (2 * fact(2-1)))) ==> 5 * (4 * (3 * (2 * fact(1))))

fact(1) ==>n * fact(n-1)，==>5 * (4 * (3 * (2 * (2-1)))) ==> 5 * (4 * (3 * (2 * fact(1))))

n = 1的是时，return 1 ，所以最终是：5 * (4 * fact(3 * fact(2 * 1)))

因此fact(1) ==>n * fact(n-1)，==>5 * (4 * (3 * (2 * (2-1)))) ==> 5 * (4 * (3 * (2 * 1)))

即 5 * 4 * 3 * 2 * 1 = 5!=120

尾递归

def fact(n):
    print(fact_iter(n, 1))
    return fact_iter(n, 1)


def fact_iter(num, product):
    if num == 1:
        return product                
    #在这里打印可以看到递归（循环）的次数
    return fact_iter(num - 1, num * product)

多表连接查询

功能：同时查询多张表里的数据

种类：inner join,left join,right join

连接方式 功能
inner join 显示两张表中，符合连接条件的记录。
left join 左表显示所有记录；右表只显示符合连接条件的记录。
right join 右表显示所有记录；左表只显示符合连接条件的记录。
  • inner join（内连接）
    功能：显示两张表中，符合连接条件的记录。

    • inner join查询的步骤

    • Left join查询的步骤

    2.观察两张表中，哪一张需要显示所有记录，要显示所有记录的表，放在 from后面。另一张表放在left join 后面。

Oracle中的数据类型

Oracle中的数据类型：字符型、数值型、日期型、LOB类型、（BLOB、CLOB）

  • 字符类型
    （1）以固定长度存放字符串：CHAR(n)、NCHAR(n)

    说明：CHAR(10)，存一个字符串 ’abc’；NCHAR以UNICODE存储每个字符

    区别：n的最大取值：char(n),n的最大取值是2000;nchar(n),n的最大取值是1000.

    （2）以可变长度存储字符串 VARCHAR2(n)、NVARCHAR2(n)

    说明：VARCHAR2(10)，存一个字符串’abc’，NVARCHAR2 以UNICODE编码存储每个字符

    区别：n的最大取值：VARCHAR2(n),n的最大取值是4000;NVARCHAR2(n),n的最大取值是2000.
  • 数值型
    从输入的数值划分：整数、小数

    （1）NUMBER类型

    a.表示小数

    举例： NUMBER(5,2)，总共5个有效数字，小数点后保留2位，例如：123.45

    b.表示整数 NUMBER(P)

    举例：NUMBER(5)，5位数的整数
  • 日期型
    • （1）DATE型

      格式：YYYY-MM-DD HH:MM:SS

      精确程度：秒
    • （2）TIMESTAMP

      精确程度：0.000001秒

      但使用较少
  • 大对象类型（Large OBject）
    • （1）BLOB类型 (Binary Large OBject)

      以二进制形式存储

      举例：图片、音频、视频

      最大存储4G字节
    • （2）CLOB类型 (Character Large Object)

      以字符串形式存储

      举例：简历、新闻、文章

      最大存储4G字节

      数据类型总结

数据 应采用的类型
字符串 Nvarchar2、varchar2
数值型 Number
日期型 Date
图片、音频、视频 BLOB
超过千字的文本 CLOB

ROWID和ROWNUM

  • ROWID
    说明：RowID可以理解为Oracle为每一张表添加的一个隐藏列。

    表中的每一行（记录），都有一个行地址，ROWID用于查询该行地址。

    任意两行的ROWID不同。

    ROWID只能查询，不能修改、删除。
  • ROWNUM
    说明：对于查询结果表，rownum能返回每一行在结果表的序号。

    结果表中的第一行rownum为1、第n行rownum为n。

配置环境变量

右键单击电脑，选择属性-高级属性-环境变量-新建，

NLS_LANG = SIMPLIFIED CHINESE_CHINA.ZHS16GBK

以上两条分开的，也就是需要新建两条记录，等号前的是变量名，等号后的是变量值

mysql ERROR 1045 (28000) Access denied for user 'ODBC'@'localhost' (using password NO)

Posted on 2020-05-01 Edited on 2022-04-14 In mysql
764 1 mins.

在cmd中敲入mysql后，出现下面的错误，找了很久没有找到解决方案

mysql ERROR 1045 (28000) Access denied for user ‘ODBC‘@’localhost’ (using password NO)

在找答案的过程中受一位网友的启发，翻译了这句话，尝试了一下，惊喜地发现困扰了我好几天的问题解决了

不过好像这个问题解决和不解决都没什么关系，我只是单纯强迫症犯了和这个问题刚起来了

这句话翻译过来就是你的mysql没有一个不用输入密码就可以登录的ODBC用户，解决方案就是：创建一个没有密码的ODBC用户，如下所示：

CREATE USER 'ODBC'@'localhost' IDENTIFIED BY '';
然后更新一下权限
flush privileges;

说到创建用户，版本不同命令是有些微差别的，如果上面这条命令不适用你的mysql，就去找找其他的创建用户的命令，这个问题的解决方案就到这里啦～～
附上图片：

另外附上mysql登录命令：mysql -u root -p;
关闭命令：service mysqld stop;
修改密码命令：set password for 'root'@'localhost'=password('991007');
赋予所有权限：grant all privileges on *.* to 'root'@'%' identified by 'root';
更新权限命令update mysql.user set Grant_priv='Y',Super_priv='Y' where user = 'root' and host = '%';
